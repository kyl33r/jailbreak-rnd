# J2: Jailbreaking to Jailbreak

## Metadata

- **Title:** Jailbreaking to Jailbreak
- **Authors:** Jeremy Kritz, Vaughn Robinson, Robert Vacareanu, Bijan Varjavand, Michael Choi, Bobby Gogov, Scale Red Team, Summer Yue, Willow E. Primack, Zifan Wang
- **ArXiv:** arXiv:2502.09638
- **Date:** February 2025
- **Type:** Attack Paper (Red Teaming)

## Summary

Uses capable LLMs to red team other models by turning refusal-trained models into effective jailbreakers. First to introduce concept of "jailbreaking to jailbreak."

## Key Insights

1. **Novel Concept**: First work introducing "jailbreaking to jailbreak" (J2) approach.

2. **LLM-of-LLMs**: Uses refusal-trained capable models as attackers.

3. **Black-Box Effective**: Works against API-only models.

4. **Scalable**: Mimics human red teaming.

## Research Approach

### J2 Methodology

1. **Seed**: Start with capable but refusal-trained model
2. **Reframe**: Repurpose refusal capabilities for attack
3. **Deploy**: Use to attack other models
4. **Iterate**: Refine based on results

## Tested Targets

- GPT-4o
- Claude-3.5-Sonnet-0620
- DeepSeek-R1
- Others

## Key Findings

- J2 prompts work on most model endpoints
- Exceptions: Claude-3.5-Sonnet-0620, DeepSeek-R1 show some resistance
- Red teaming effectiveness demonstrated

## Defenses Challenged

- Refusal training
- Output filtering
- Safety guardrails

## Implications

- Refusal training doesn't prevent attack capability
- Models can be repurposed for attacks
- Need for stronger safeguards

## Key Finding

A model trained to refuse harmful requests can still be used to generate effective jailbreak prompts for attacking other models.
